# Cloudflare Origin Certificate Setup Guide

This guide explains how to set up Cloudflare Origin Certificates for scalable SSL management across thousands of domains.

## Why Origin Certificates?

- **Unlimited Certificates**: One certificate can cover thousands of domains
- **No Rate Limits**: Unlike Let's Encrypt (50 certs/week), no limits
- **Free**: Included with Cloudflare accounts
- **Scalable**: Perfect for multi-tenant platforms
- **Long Validity**: 15-year certificates (auto-renewed by Cloudflare)

## Setup Steps

### 1. Create Origin Certificate in Cloudflare Dashboard

1. Log in to [Cloudflare Dashboard](https://dash.cloudflare.com)
2. Select your zone (e.g., `smartstore.ng`)
3. Go to **SSL/TLS** → **Origin Server**
4. Click **Create Certificate**

### 2. Configure Certificate

**Option A: Wildcard Certificate (Recommended for Base Domain)**
- **Hostnames**: `smartstore.ng`, `*.smartstore.ng`
  - `smartstore.ng` - Base domain (explicitly required, NOT covered by wildcard)
  - `*.smartstore.ng` - All subdomains (covers: `api.smartstore.ng`, `app.smartstore.ng`, `www.smartstore.ng`, and all user subdomains)
- **Validity**: 15 years
- **Private Key Type**: RSA (2048) or ECDSA (P-256)
- Click **Create**

**Important**: The wildcard `*.smartstore.ng` covers:
- ✅ `api.smartstore.ng` (API service)
- ✅ `app.smartstore.ng` (Web/Dashboard service)
- ✅ `www.smartstore.ng` (www subdomain)
- ✅ `acme.smartstore.ng` (user sites)
- ✅ Any other subdomain
- ❌ `smartstore.ng` (base domain - must be added explicitly)

**Option B: Multi-Domain Certificate (For Custom Domains)**

Multi-domain certificates are useful when you have many custom domains (user's own domains) that aren't subdomains of your base domain.

**Steps:**
1. In the **Hostnames** field, add multiple domains, one per line or separated by commas:
   ```
   acmecorp.com
   www.acmecorp.com
   shop.example.com
   blog.example.com
   *.example.com
   anothersite.com
   www.anothersite.com
   ```

2. **Important**: You can add up to 50 hostnames per certificate
   - Individual domains: `acmecorp.com`, `shop.example.com`
   - Wildcards: `*.example.com` (covers all subdomains of example.com)
   - Both www and non-www: `acmecorp.com`, `www.acmecorp.com`

3. **Validity**: 15 years
4. **Private Key Type**: RSA (2048) or ECDSA (P-256)
5. Click **Create**

**Best Practices:**
- Group related domains together (e.g., all domains for one customer)
- Use wildcards when possible (e.g., `*.example.com` instead of listing each subdomain)
- Create separate certificates for different customer groups if needed
- You can create multiple Origin Certificates for different domain groups

**Example Multi-Domain Certificate:**
```
Hostnames:
- acmecorp.com
- www.acmecorp.com
- shop.example.com
- blog.example.com
- *.example.com
- anothersite.com
- www.anothersite.com
```

This single certificate covers all listed domains and all subdomains of `example.com`.

### 3. Download Certificate and Private Key

1. After creating the certificate, you'll see:
   - **Origin Certificate** (the certificate)
   - **Private Key** (the private key)

2. **Copy both** to your server

### 4. Install on Nginx Server

**Create directories:**
```bash
mkdir -p /etc/ssl/smartstore/certs
mkdir -p /etc/ssl/smartstore/keys
```

**Save certificate:**
```bash
# Save the Origin Certificate
cat > /etc/ssl/smartstore/certs/cloudflare-origin.crt << 'EOF'
-----BEGIN CERTIFICATE-----
[Paste your Origin Certificate here]
-----END CERTIFICATE-----
EOF
```

**Save private key:**
```bash
# Save the Private Key
cat > /etc/ssl/smartstore/keys/cloudflare-origin.key << 'EOF'
-----BEGIN PRIVATE KEY-----
[Paste your Private Key here]
-----END PRIVATE KEY-----
EOF
```

**Set permissions:**
```bash
chmod 644 /etc/ssl/smartstore/certs/cloudflare-origin.crt
chmod 600 /etc/ssl/smartstore/keys/cloudflare-origin.key
```

### 5. Configure Environment Variables

Add to your `.env` file:

```env
# Cloudflare Origin Certificate
CLOUDFLARE_USE_ORIGIN_CERT=true
CLOUDFLARE_ORIGIN_CERT_PATH=/etc/ssl/smartstore/certs/cloudflare-origin.crt
CLOUDFLARE_ORIGIN_KEY_PATH=/etc/ssl/smartstore/keys/cloudflare-origin.key
CLOUDFLARE_SSL_MODE=full  # or 'strict' for maximum security
```

### 6. Update Nginx Configuration

The Origin Certificate will be automatically used in Nginx configs generated by the API service.

**For base domains** (`smartstore.ng`, `*.smartstore.ng`), update `nginx/nginx.conf.template`:

```nginx
ssl_certificate /etc/ssl/smartstore/certs/cloudflare-origin.crt;
ssl_certificate_key /etc/ssl/smartstore/keys/cloudflare-origin.key;
```

**For custom domains**, the API service automatically generates Nginx configs with the Origin Certificate paths.

### 7. Verify Setup

**Check certificate:**
```bash
openssl x509 -in /etc/ssl/smartstore/certs/cloudflare-origin.crt -text -noout
```

**Test Nginx config:**
```bash
nginx -t
```

**Reload Nginx:**
```bash
nginx -s reload
```

## How It Works

### For Base Domains (`smartstore.ng`, `*.smartstore.ng`)

1. **One Wildcard Certificate** covers:
   - `smartstore.ng` (base domain - explicitly required)
   - `*.smartstore.ng` (all subdomains - wildcard)
2. The wildcard `*.smartstore.ng` covers:
   - ✅ `api.smartstore.ng` (API service)
   - ✅ `app.smartstore.ng` (Web/Dashboard service)
   - ✅ `www.smartstore.ng` (www subdomain)
   - ✅ All user subdomains (e.g., `acme.smartstore.ng`)
   - ❌ `smartstore.ng` (base domain - must be added explicitly)
3. Installed once on Nginx server
4. Works for unlimited subdomains
5. Cloudflare validates the certificate when proxying traffic

### For Custom Domains (User's Own Domains)

1. **Multi-Domain Certificate** can cover many custom domains
2. Or use **Universal SSL** (automatic per-domain)
3. Origin Certificate is validated by Cloudflare
4. Nginx uses the same certificate for all domains

## SSL Modes

### Full (Recommended)
- **Cloudflare ↔ User**: Cloudflare certificate (automatic)
- **Cloudflare ↔ Nginx**: Validates Origin Certificate
- **End-to-end encryption**: ✅
- **Requires**: Origin Certificate on Nginx

### Full (Strict)
- **Cloudflare ↔ User**: Cloudflare certificate (automatic)
- **Cloudflare ↔ Nginx**: Validates Origin Certificate (strict validation)
- **End-to-end encryption**: ✅
- **Maximum security**: ✅
- **Requires**: Origin Certificate on Nginx

### Flexible
- **Cloudflare ↔ User**: Cloudflare certificate (automatic)
- **Cloudflare ↔ Nginx**: HTTP (no SSL)
- **End-to-end encryption**: ❌ (only to Cloudflare)
- **Requires**: No certificate on Nginx
- **Not recommended** for production

## Troubleshooting

### Certificate Not Found

**Error**: `Cloudflare Origin Certificate not found`

**Solution**:
1. Verify certificate files exist at configured paths
2. Check file permissions (cert: 644, key: 600)
3. Verify environment variables are set correctly

### Domain Not Proxied

**Error**: `Domain is not proxied. Origin certificate requires Cloudflare proxy.`

**Solution**:
1. Go to Cloudflare Dashboard → DNS
2. Find the domain's A record
3. Enable proxy (orange cloud icon)
4. Wait for DNS propagation

### SSL Mode Mismatch

**Error**: SSL not working despite certificate installed

**Solution**:
1. Ensure SSL mode is set to `full` or `strict` in Cloudflare
2. Verify Origin Certificate is installed on Nginx
3. Check Nginx config uses correct certificate paths

## Best Practices

1. **Use Wildcard Certificates** for base domains (`*.smartstore.ng`)
2. **Use Multi-Domain Certificates** for custom domains (group related domains)
3. **Set SSL Mode to `full` or `strict`** for maximum security
4. **Keep Private Keys Secure** (600 permissions, restricted access)
5. **Monitor Certificate Expiration** (15 years, but monitor anyway)
6. **Use Separate Certificates** for different domain groups if needed

## Scaling to Thousands of Domains

### Strategy 1: One Wildcard Certificate
- Create wildcard certificate: `*.smartstore.ng`
- Install once on Nginx
- Works for unlimited subdomains
- **Best for**: Base domain subdomains

### Strategy 2: Multiple Multi-Domain Certificates
- Create multiple certificates, each covering 100+ domains
- Install all certificates on Nginx
- API service selects appropriate certificate
- **Best for**: Custom domains

### Strategy 3: Hybrid Approach
- Wildcard certificate for base domains
- Multi-domain certificates for custom domains
- **Best for**: Mixed deployment

## Security Notes

- **Never commit certificates or keys to version control**
- **Use secure file permissions** (cert: 644, key: 600)
- **Rotate certificates periodically** (even though they're 15-year)
- **Monitor certificate usage** in Cloudflare dashboard
- **Use Full (Strict) mode** for maximum security

## API Integration

The SSL service automatically:
1. Detects if Origin Certificate is configured
2. Uses Origin Certificate if available
3. Falls back to Universal SSL if not
4. Generates Nginx configs with correct certificate paths
5. Reloads Nginx automatically

No manual intervention required after initial setup!
