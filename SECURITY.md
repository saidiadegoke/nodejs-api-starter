# Security policy

## Supported versions

Security updates are applied to the default branch (`main` or `master`, depending on the repository). Use the latest commit when deploying.

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Instead:

1. For **[nodejs-api-starter](https://github.com/saidiadegoke/nodejs-api-starter)**, use the repository **[Security](https://github.com/saidiadegoke/nodejs-api-starter/security)** tab (“Report a vulnerability” / private advisories) when enabled (see [GitHub’s documentation](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)), **or**
2. Email the maintainer: **rasheedsaidi@gmail.com** (Saidi Adegoke).

Include:

- A short description of the issue and its impact
- Steps to reproduce or proof-of-concept (if safe to share)
- Affected versions or commit range, if known

Maintainers will aim to acknowledge reports within a few business days. This template is provided as-is; production deployments remain your responsibility (secrets, TLS, rate limits, dependency updates, etc.).
